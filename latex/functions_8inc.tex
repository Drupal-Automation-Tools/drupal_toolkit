This works like the j\-Query.\-extend function.

For each key in \$extension, one of the following happens\-:
\begin{DoxyItemize}
\item If the key is numeric, the value is appended onto \$base with a new numeric key.
\item If \$base does not have the key, \$base\mbox{[}key\mbox{]} is set to the value.
\item If \$base\mbox{[}key\mbox{]} and the value are both scalar, then \$base\mbox{[}key\mbox{]} is set to the value.
\item If \$base\mbox{[}key\mbox{]} is an array and the value is scalar, then the value is appended to \$base\mbox{[}key\mbox{]} with a new numeric key.
\item If \$base\mbox{[}key\mbox{]} not an array and the value is, then \$base\mbox{[}key\mbox{]} is turned into an array (and continue with the next instruction).
\item If both of them are arrays, then this function is called recursively with \$base\mbox{[}key\mbox{]} and value as the two arguments.
\end{DoxyItemize}

This is essentially like mixing the behaviour of array\-\_\-replace() (for scalar values) and array\-\_\-merge() (for arrays).

Note that this function operates in place on \$base (much like sort()). It has no return value.
